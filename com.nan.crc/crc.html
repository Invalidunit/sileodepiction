Check repo change 配置文件配置用法

一、从 App Store 安装 Bark

1.请先从 App Store 中安装 Bark App，这是为你推送通知的方式，主要目前还没找到（也可能我不会）纯终端可以往越狱设备推送通知的命令

打开 Bark 安装页

点击上方跳转 App Store

二、获取 Bark 推送 key

1.打开 Bark app 后点击中心的圆形注册按钮

2.按下图黄色框框圈起来的位置复制链接

3.将复制出来的链接仅保留上图红色框框部分（两个斜杠中间夹着的部分就是 key，不包含斜杠）

三、修改配置文件中关于 bark 的配置

1.从 Filza 中打开文件

rootfull

/var/mobile/Library/Preferences/com.nan.crc.plist

rootless

/var/jb/var/mobile/Library/Preferences/com.nan.crc.plist

2.点击 bark 展开子菜单，找到 key 也就是下图红框位置

3.点击 key 右侧的 i 也就是上图黄框位置，并粘贴第二步中获得的 key

4.配置完 key 后可以重装软件包，这会触发一条测试用的通知，告诉你开始工作了

扩展

一、添加新的检测源

1.从 Filza 中打开文件

rootfull

/var/mobile/Library/Preferences/com.nan.crc.plist

rootless

/var/jb/var/mobile/Library/Preferences/com.nan.crc.plist

2.点击 source 展开子菜单，按照下图格式新建 Item

主要构成部分为 enable 和 url

enable 用来控制是否为这个源启用检测

url 用来为 crc 提供源地址

二、自定义通知标题

1.从 Filza 中打开文件

rootfull

/var/mobile/Library/Preferences/com.nan.crc.plist

rootless

/var/jb/var/mobile/Library/Preferences/com.nan.crc.plist

2.点击 bark 展开子菜单，找到 title，下图红框部分

点击上图黄框部分可以换成任何你想自定义的文字，软件源名称可以用 repo_name 来代替

例如：

repo_name 已变更

发现 repo_name 已变动

三、自定义部分通知内容

1.从 Filza 中打开文件

rootfull

/var/mobile/Library/Preferences/com.nan.crc.plist

rootless

/var/jb/var/mobile/Library/Preferences/com.nan.crc.plist

2.点击 bark 展开子菜单，找到 body，如下图红框部分

里面会有 3 个子项目，分别是 add（新增），del（移除），up（更新现有版本）

按上图的黄框部分可以将他们换成你喜欢的语句，不过目前通知顺序改不了

他们在通知横幅中的顺序从上到下依次是“更新”，“新增”，“删除”
